# Migration slice records

Create one Markdown record per proposed slice. Copy this checklist; do not change YARP until its state is `approved`.

```markdown
# Slice: <name>

- Manifest route IDs:
- Boundary and session owner (including pre-authentication OIDC state):
- Application/framework session purposes:
- Anonymous attribution/cookie owner or explicit exclusion:
- Risk-register IDs:
- Rails evidence:
- Contract fixtures:
- Browser/visual/accessibility evidence:
- Database and side-effect evidence:
- Known differences:
- Rollback route change:
- Reviewer/approval:
- State: proposed | implementing | parity-proven | approved | rolled-back
```
